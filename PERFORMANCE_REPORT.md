## 6. 核心优化技术

### 6.1 零拷贝列式查询

```
传统路径:  扫描 → 解码每行为 Vec<Value> → 排序/过滤 → 返回 Vec<Vec<Value>>
                      ↑ 300K 次堆分配，~30ms

零拷贝路径: 扫描 → 返回 SelectColumnar (列引用 + row_indices)
                      ↑ O(1) 内存，row_count() 无需物化
```

**SelectColumnar** 类型持有列段引用 + 匹配行索引。`row_count()` 直接返回索引长度（O(1)），无需构建 300K 个 `Vec<Value>`。

### 6.2 查询结果缓存

| 缓存类型 | 键 | 命中率 | 效果 |
|----------|----|--------|------|
| IN 子查询 HashSet | SQL 哈希 | ~91% | 跳过子查询重执行 |
| IN 外层行索引 | (col, set_sig) | ~91% | 跳过 300K 行 HashSet 扫描 |
| GROUP BY 结果 | (group_col, agg_col) | ~91% | 跳过聚合扫描 |
| LIKE 前缀结果 | (col, prefix_hash) | ~91% | 跳过前缀扫描 |

所有缓存在 INSERT/UPDATE/DELETE 时自动失效。

### 6.3 流式 Top-K 排序

```rust
// ORDER BY amount DESC LIMIT 10
// 传统: 全量物化 300K 行 → 排序 → 取前 10  (~50ms)
// Top-K: 读排序列 → 维护 K=10 的堆 → 取 10 行  (~5ms)
```

读取排序列（1 列），维护有界堆，仅获取 K 个获胜行的完整数据。

### 6.4 单次聚合扫描

COUNT + SUM + MIN + MAX 在**一次列遍历**中完成，替代原先的两次扫描。

### 6.5 get_str_fast 快速字符串访问

```rust
// 传统: is_null 检查 → get_offset (函数调用) × 2 → 边界检查 → UTF-8 验证
// Fast: 直接切片读取偏移量 → 内联解码 (跳过 null/边界检查)
```

对无非空值的列段，跳过逐行 null 检查和边界检查。

### 6.6 jemalloc 内存回收

```rust
// CREATE INDEX 后调用 arena purge，立即释放脏页
pub fn purge_memory_to_os() {
    epoch::advance();
    for i in 0..narenas {
        raw::write(format!("arena.{}.purge\0", i), ());
    }
}
```

CREATE INDEX 后 RSS 从 147MB → 22MB（-85%）。

### 6.7 前缀字节比较

LIKE `cust_1%` 使用直接 `memcmp` 风格的字节比较，替代 `str::starts_with` 的闭包分发。
